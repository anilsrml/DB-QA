Bu proje, kullanıcıların mevcut bir PostgreSQL veritabanını doğal dil kullanarak sorgulayabilmesini sağlayan bir yapay zeka agent sistemi geliştirmeyi amaçlamaktadır. Veritabanı, proje kapsamında Python ile oluşturulmak zorunda değildir; sistem, hali hazırda var olan ve başka uygulamalar tarafından kullanılan bir PostgreSQL veritabanına bağlanacak şekilde tasarlanmıştır. Böylece çözüm, gerçek dünya kurumsal sistemleriyle uyumlu bir yapı sunar.

Sistemin merkezinde bir agent katmanı yer alır. Bu agent, kullanıcıdan gelen doğal dildeki sorguyu alır ve önce sorgunun amacını anlamaya çalışır. Ardından veritabanına ait tablo ve kolon yapıları (schema bilgisi) kullanılarak, bu sorguya karşılık gelen SQL ifadesinin nasıl olması gerektiğine karar verir. SQL ifadesinin üretimi, büyük dil modeli (LLM) tarafından gerçekleştirilir. Ancak LLM, serbest şekilde SQL yazmaz; yalnızca agent tarafından kendisine sağlanan tablo, kolon ve kural bilgileri çerçevesinde SQL üretir.

Üretilen SQL sorgusu doğrudan veritabanına gönderilmeden önce agent tarafından güvenlik ve doğruluk açısından kontrol edilir. Bu aşamada sadece SELECT türünde sorgulara izin verilir; veri silme, güncelleme veya veritabanı yapısını değiştirmeye yönelik ifadeler engellenir. Kontrolden geçen sorgu PostgreSQL veritabanına gönderilir ve elde edilen sonuçlar agent tarafından işlenerek kullanıcıya anlaşılır bir metin halinde sunulur.

Bu mimaride PostgreSQL, pasif bir veri kaynağıdır ve sistemin yapay zeka bileşenlerinden haberdar değildir. Agent sistemi, veritabanına standart bir uygulama gibi bağlanır ve yalnızca okuma işlemleri gerçekleştirir. Yapay zeka ise doğrudan veriyi yorumlamak yerine, kullanıcının niyetini doğru SQL sorgusuna dönüştüren bir çevirmen rolü üstlenir. Bu sayede sistem hem esnek doğal dil sorgularını destekler hem de güvenli ve kontrol edilebilir bir şekilde çalışır.
